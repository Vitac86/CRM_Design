Краткое резюме

content-116.phtml — это страница списка комплаенса. Он совмещает: вывод клиентов со статусом 3 («На проверке»), все бизнес-действия (одобрить / на доработку / заблокировать) и inline-просмотр данных через модальные окна UIKit. Отдельной «карточки» клиента в старом коде не существует.

Новый дизайн разделяет функциональность на две страницы: реестр (compliance.html) и карточку проверки (compliance-card.html). Это архитектурное изменение — главный вызов миграции: вся бизнес-логика из phtml переходит в content_custom, inline-модалки заменяются переходом на отдельную страницу.

✅ Подтверждено из старого кода status_klienta (3/4/5/−1), hash, basicname, inn, num_investika, surname/namecl/patronymic, komplajns_zamechaniya, все 7 обработчиков действий, DualLog
❌ Не найдено в старом коде Резидентство, подтип ЮЛ, ИП-тип, KYC/AML статусы, документы, риск-уровень, контакты, офицер, дата обновления, черновик, экспорт CSV

LIST public function getComplianceList($params = [])

Назначение: Возвращает список клиентов для реестра со всеми статусами.

Источник: getFirmsIdByStatus() / getUserIdByStatus() (стр. 67–82) + HTML-секция списка (536–748).

Риск: UmiCMS selector работает с одним типом. Два запроса (firm + phys) + PHP-merge. Глобальная сортировка и точный offset нетривиальны.

]

Риски и вопросы к владельцу проекта

Технические риски

  • Р1
    Два selector'а вместо одного. UmiCMS selector работает с одним типом объекта. Объединение firm + phys требует двух запросов и PHP-merge. Глобальная сортировка и точный offset нетривиальны — возможно снижение производительности.
  • Р2
    Тип данных status_klienta. 4 (int), "5" (string), "-1" (string) — несогласованность. Стандартизировать до int перед миграцией.
  • Р3
    $legal_point_check_info_firms / $legal_point_check_info_phys не определены. Используются в строках 617, 727, но нигде в файле не объявлены. Источник (вероятно initialize.php) неизвестен. Без них блок «Подробнее» пуст.
  • Р4
    Старые GET-обработчики нельзя удалять досрочно. ?checked=, ?checked_phys=, ?redact_phys= — прямые ссылки в HTML. До полного переключения на новый UI должны работать.
  • Р5
    ~60% полей карточки без источника. Начинать реализацию карточки без инвентаризации UMI-полей — риск переделки.
  • Р6
    Черновик и CSV — полностью новые функции без прецедента в старом коде. Объём не включён в оценки.
